Wrangler 提供了许多命令来管理您的 Cloudflare Workers。#

#docs#

在默认浏览器中打开 Cloudflare 开发人员文档。

wrangler docs [<COMMAND>]

COMMAND string 自选

要了解详细信息的 Wrangler 命令。这将打开默认浏览器，进入描述该命令的文档部分。

#init#

通过 create-cloudflare-cli （C3） 工具创建新项目。有多种 Web 框架和模板可供选择。默认情况下会安装依赖项，并可选择立即部署项目。

wrangler init [<NAME>] [OPTIONS]

NAME string 可选（默认：工作目录的名称）

Workers 项目的名称。这是生成 wrangler.toml 的配置文件中的目录名称和 name 属性。

–yes boolean 自选

对新项目的任何提示回答“是”。

–from-dash string 自选

获取从仪表板初始化的 Worker。这是通过传递标志和 Worker 名称来完成的

wrangler init --from-dash <WORKER_NAME>

使用该命令后，该 --from-dash 命令不会自动同步对仪表板所做的更改。因此，建议您继续使用 CLI。

#generate#

使用现有 Workers 模板创建新项目。

wrangler generate [<NAME>] [TEMPLATE]

NAME string 可选（默认：工作目录的名称）

Workers 项目的名称。这是生成 wrangler.toml 的配置文件中的目录名称和 name 属性。

TEMPLATE string 自选

GitHub 模板的 URL，默认的 worker-template.

浏览 cloudflare/workers-sdk 上的可用模板列表存储库。

#d1#

与 Cloudflare 的 D1 服务进行交互。

create#

创建新的 D1 数据库，并提供将放入 wrangler.toml 文件中的绑定和 UUID。

wrangler d1 create <DATABASE_NAME> [OPTIONS]

DATABASE_NAME string 必填

新 D1 数据库的名称。

–location string 自选

为数据库服务提供可选位置提示。 可用选项包括 weur（西欧）、eeur（东欧）、apac（亚太地区）、oc（大洋洲）、wnam（北美西部）和 enam（北美东部）。
